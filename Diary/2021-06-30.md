2. 中心の周辺となるエンティティはwhen→「出金日」, where →「出金先」, what→「出金明細」
- 出金
	- id
	- 出金日(出金エンティティにまとめられる)
- 出金先
	- id
	- 名前
- 出金明細
	- id
	- コード
	- 勘定科目
	- 摘要
	- 金額

3. この場合、{ id } → { コード } , { コード } → { 勘定科目 } で推移的関数従属になっているので、新しくテーブルで分け、出金明細エンティティにコード(FK)で外部キーを追加。
4. 出金エンティティに誰に対して出金したかも関連づける。出金明細エンティティにいつ、誰に対して出勤したかも関連づける。
